Function glossary by module group

Bootstrap

Bootstrap Core

  • _init_bootstrap — Initialize the early bootstrap environment before core library loading.
  • _parse_bootstrap_args — Parse framework-level bootstrap switches before script/builtin parsing.
  • sgnd_bootstrap — Initialize the Testadura framework runtime environment.
  • sgnd_check_license — Verify that the current framework license has been accepted.
  • sgnd_disable_save_state — Disable automatic state persistence.
  • sgnd_enable_save_state — Enable automatic state persistence.
  • sgnd_load_ui_style — Load the active UI palette and style definitions.
  • sgnd_module_init_metadata — Define module-scoped metadata variables for a sourced library and
  • sgnd_on_exit_add — Register a new EXIT handler to be executed by the EXIT dispatcher.
  • sgnd_on_exit_install — Install the framework EXIT dispatcher (trap) exactly once per process.
  • _sgnd_on_exit_run — Execute registered EXIT handlers (LIFO) while preserving the original exit code.
  • sgnd_parse_statespec — Parse a pipe-delimited state specification into component fields.
  • sgnd_save_state — Persist selected state variables to storage (as configured by SGND_STATE_VARIABLES).
  • _sgnd_save_state_dispatch — Save state on EXIT only for clean exits.
  • sgnd_script_init_metadata — Initialize executable metadata from structured header comments.
  • _source_corelibs — Source core framework libraries in the defined order (SGND_CORE_LIBS).
  • _source_usinglibs — Source optional script-requested libraries from SGND_COMMON_LIB.

Bootstrap Environment

  • _build_framework_dirs — Rebuild the canonical list of framework directories from current path globals.
  • sgnd_apply_defaults — Apply default values for bootstrap globals if currently unset.
  • sgnd_defaults_reset — Reset framework-global configuration variables to their default state.
  • sgnd_ensure_dirs — Ensure that framework directories exist and have appropriate ownership.
  • sgnd_rebase_directories — Recompute framework, user, and script-scoped paths from current root settings.
  • sgnd_rebase_framework_cfg_paths — Derive framework-global cfg file paths from current cfg directories.

  • sgnd_cfg_unset — Remove a config key from the file and unset it in the current shell.

  • _sgnd_kv_reset_file — Hard-delete a KEY=VALUE file.
  • _sgnd_kv_set — Write or update a KEY=VALUE entry in a file.
  • _sgnd_kv_unset — Remove a KEY=VALUE entry from a file.
  • sgnd_state_get — Read a state value from the state file.
  • sgnd_state_has — Test whether a state key exists in the state file.
  • sgnd_state_list_keys — List keys currently present in the state file.
  • sgnd_state_load — Load the state file into shell variables.
  • sgnd_state_load_keys — Load selected state keys from the state store into shell variables.
  • sgnd_state_reset — Hard-reset the state file by deleting it.
  • sgnd_state_save_keys — Persist a list of shell variables to the state store.
  • sgnd_state_set — Persist a state KEY=VALUE pair and update the current shell variable.
  • sgnd_state_unset — Remove a state key from the file and unset it in the current shell.
