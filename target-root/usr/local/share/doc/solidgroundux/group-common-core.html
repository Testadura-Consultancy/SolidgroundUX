1 Argument Parsing — Parse and manage command-line arguments for scripts
  • _sgnd_arg_find_spec — Find the effective argument specification that matches a long or short option token. [private]
  • _sgnd_arg_init_defaults — Initialize option variables and build the effective argument specification list. [private]
  • _sgnd_arg_split — Split a single SGND_ARGS_SPEC definition into internal scratch variables. [private]
  • _sgnd_arg_validate_enum — Validate whether a value is present in a comma-separated enum choice list. [private]
  • sgnd_builtinarg_handler — Apply framework-defined built-in argument behavior after argument parsing. [public]
  • sgnd_parse_args — Parse command-line arguments according to the effective argument specification. [public]
  • sgnd_show_help — Render and display the generated help text for the current script. [public]
2 Configuration Management — Load, manage, and persist configuration settings
  • sgnd_cfg_domain_apply — Apply configuration for a domain from system and user cfg files. [public]
  • sgnd_cfg_ensure_files — Ensure required config files exist for a domain based on its specs. [public]
  • sgnd_cfg_get — Read a config value from the config file. [public]
  • sgnd_cfg_has — Test whether a config key exists in the config file. [public]
  • sgnd_cfg_has_audience — Test whether a cfg spec array contains entries for a requested audience. [public]
  • sgnd_cfg_load — Load a config file into shell variables. [public]
  • sgnd_cfg_load_file — Load a specific cfg file into shell variables for domain-level processing. [public]
  • sgnd_cfg_reset — Hard-reset the config file by deleting it. [public]
  • sgnd_cfg_set — Persist a config KEY=VALUE pair and update the current shell variable. [public]
  • sgnd_cfg_show_keys — Display selected config keys and their stored values. [public]
  • sgnd_cfg_unset — Remove a config key from the file and unset it in the current shell. [public]
  • sgnd_cfg_warn_missing_syscfg — Emit a warning when a required system cfg file is missing. [public]
  • sgnd_cfg_write_skeleton_filtered — Write an auto-generated cfg skeleton filtered by audience. [public]
  • _sgnd_is_ident — Test whether a string is a valid shell identifier. [private]
  • _sgnd_kv_get — Read a value for a key from a KEY=VALUE file. [private]
  • _sgnd_kv_has — Test whether a KEY exists in a KEY=VALUE file (even if empty). [private]
  • _sgnd_kv_list_keys — Emit file contents as '#39;key [value'#39; lines (order preserved).|private]
  • _sgnd_kv_load_file — Load KEY=VALUE pairs from a file into shell variables. [private]
  • _sgnd_kv_reset_file — Hard-delete a KEY=VALUE file. [private]
  • _sgnd_kv_set — Write or update a KEY=VALUE entry in a file. [private]
  • _sgnd_kv_unset — Remove a KEY=VALUE entry from a file. [private]
  • sgnd_state_get — Read a state value from the state file. [public]
  • sgnd_state_has — Test whether a state key exists in the state file. [public]
  • sgnd_state_list_keys — List keys currently present in the state file. [public]
  • sgnd_state_load — Load the state file into shell variables. [public]
  • sgnd_state_load_keys — Load selected state keys from the state store into shell variables. [public]
  • sgnd_state_reset — Hard-reset the state file by deleting it. [public]
  • sgnd_state_save_keys — Persist a list of shell variables to the state store. [public]
  • sgnd_state_set — Persist a state KEY=VALUE pair and update the current shell variable. [public]
  • sgnd_state_unset — Remove a state key from the file and unset it in the current shell. [public]
