Advance content type after emitted headers
SolidgroundUX / doc-processor.sh / Local definitions / Action / _guess_nextcontenttype
Purpose
Prepare the parser for body content following a recognized header line.
Behavior
- Maps module, section, item, and recognized headers to their body content types.
- Leaves unknown or body content types unchanged.
Inputs (globals):
doc_contenttype
Outputs (globals):
doc_contenttype
Returns
0 always.
Usage
_guess_nextcontenttype
